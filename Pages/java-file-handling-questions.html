Java File Handling: Data Persistence and I/O Streams Guide

This easy Java file handling tutorial covers file handling basics in Java, including how to read and write files, Java I/O streams, and real-world examples. If you are preparing for interviews or learning Java for the first time, this guide will help you understand file handling in Java step-by-step.

Simple Java File Reading Example (Easy)

import java.io.*;

public class ReadFileExample {
    public static void main(String[] args) {
        try {
            BufferedReader br = new BufferedReader(new FileReader("data.txt"));
            String line;
            while((line = br.readLine()) != null) {
                System.out.println(line);
            }
            br.close();
        } catch(Exception e) {
            System.out.println(e);
        }
    }
}

What is File Handling? In Java, file handling is the process of reading data from a file and writing data to a file. Java uses the concept of a Stream—a sequence of data—to make I/O operations fast and efficient. Whether you are reading a simple text file or a complex binary image, the principles of Streams remain the same.

1. The Stream Hierarchy: Byte vs. Character

Java defines two types of streams to handle different kinds of data. Knowing when to use which is a common interview checkpoint.

A. Byte Streams

Byte streams are used to handle input and output of 8-bit bytes. They are primarily used for binary data like images, audio, or video files. The main classes are FileInputStream and FileOutputStream.

B. Character Streams

Character streams are used to handle 16-bit Unicode characters. They are specialized for reading and writing text files. The main classes are FileReader and FileWriter. Character streams automatically handle character encoding, making them more suitable for internationalized text.

2. Core Classes in java.io

The java.io package contains several classes that perform various file-related tasks. Below is a comparison of the most frequently used classes.

Class Purpose I/O Type
File Used to create, delete, and inspect file/directory metadata. Metadata
FileInputStream Reads raw bytes from a file. Byte
BufferedReader Reads text from a character-input stream, buffering characters for efficiency. Character
PrintWriter Prints formatted representations of objects to a text-output stream. Character

The File class is an abstract representation of file and directory pathnames. It is used to perform operations like checking if a file exists, creating a new directory, or getting the file size.

4. Top File Handling Interview Questions

Q1. What is Serialization and Deserialization?

Serialization is the process of converting an object's state into a byte stream so it can be saved to a file or sent over a network. Deserialization is the reverse process where the byte stream is used to recreate the actual Java object in memory. To make a class serializable, it must implement the Serializable marker interface.

Q2. What is the 'transient' keyword in Serialization?

The transient keyword is used with variables that you do not want to be serialized. For example, if a class contains a sensitive field like a password, marking it as transient ensures its value is not saved during the serialization process.

Q3. Why is it better to use BufferedReader than FileReader directly?

BufferedReader reads a large chunk of data from the file and stores it in an internal buffer. This reduces the number of I/O operations (accessing the disk), which is much faster than reading character by character using FileReader.

Q4. What is the difference between Path and File in Java NIO.2?

Introduced in Java 7, Path is more powerful and flexible than File. It supports symbolic links and provides better error handling. The Files utility class works with Path objects to perform massive file operations (like copying an entire directory) with single lines of code.

Q5. How do you append text to an existing file?

When creating a FileWriter or FileOutputStream, you can pass a boolean true as the second argument to the constructor: new FileWriter("file.txt", true). This opens the file in append mode instead of overwriting it.

5. Advanced Topic: Java NIO (New I/O)

Modern Java applications often use the java.nio package for high-performance file handling. Unlike standard I/O which is block-oriented and synchronous, NIO uses Channels and Buffers.

  • Channel: A medium through which data is transported (like a pipe).
  • Buffer: A container for data that is being moved.
  • Non-blocking: NIO allows a thread to ask a channel to read data into a buffer and continue doing other tasks until the data is ready.
