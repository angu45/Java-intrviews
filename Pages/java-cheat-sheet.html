02. OOP Pillars (Object Oriented Programming)

Java is a robust OOP language. Understanding these four pillars is non-negotiable for any developer.

1. Encapsulation

Wrapping data (variables) and code (methods) together as a single unit.

class User {
    private String name; // Private Variable
    public String getName() { return name; } // Getter
    public void setName(String n) { name = n; } // Setter
}

2. Inheritance

A mechanism where one class acquires the properties of another.

class Animal { void eat() { } }
class Dog extends Animal { 
    void bark() { } // Dog inherits eat()
}

3. Polymorphism

One name, many forms. Method Overloading vs Overriding.

// Overloading: Same name, different params
void add(int a, int b) { }
void add(int a, int b, int c) { }

4. Abstraction

Hiding implementation details and showing only functionality.

abstract class Shape {
    abstract void draw(); // No body
}

04. Java 8+ Streams & Lambda

⚡ Modern Functional Programming

List<String> names = Arrays.asList("Java", "Python", "JavaScript", "C++");

// Filter, Map and Collect
List<String> filtered = names.stream()
    .filter(s -> s.startsWith("J"))      // Filter elements
    .map(String::toUpperCase)            // Transform elements
    .sorted()                             // Sort
    .collect(Collectors.toList());        // Collect to List

System.out.println(filtered); // [JAVA, JAVASCRIPT]
Note: Streams are lazy. They don't process data until a terminal operation (like collect, forEach, or reduce) is called.

05. Exception Handling & Best Practices

Standard Try-Catch

try {
    int data = 100/0;
} catch (ArithmeticException e) {
    System.out.println("Error: " + e);
} finally {
    System.out.println("I always run.");
}

Custom Exception

class MyEx extends Exception {
    public MyEx(String msg) {
        super(msg);
    }
}
// Throw it:
throw new MyEx("User not found");

🔥 Quick Interview Revision Checklist

  • ✔ JDK vs JRE vs JVM: JVM runs code; JRE = JVM + Libs; JDK = JRE + Tools.
  • ✔ Immutable Strings: Strings cannot be changed once created. Use StringBuilder for heavy edits.
  • ✔ Static Keyword: Belongs to the class, not the instance.
  • ✔ HashMap working: Works on hashing principle using hashCode() and equals().
  • ✔ Interface vs Abstract: Interface is 100% abstract (pre-Java 8); Abstract can have state.
  • ✔ Garbage Collection: Automatic memory management using Mark-and-Sweep.

❓ Frequently Asked Java Questions

Q. What is the best way to revise Java quickly for an interview?

The best way is to focus on core concepts: OOP pillars, Collections framework, and Exception handling. Use this cheat sheet to brush up on syntax and solve at least 5-10 coding problems on Strings and Arrays.

Q. Why Java is not a purely object-oriented language?

Java supports Primitive Data Types (int, char, etc.) which are not objects. Hence, it is not 100% object-oriented.

Q. Is Java still relevant in 2026?

Absolutely. Java remains the backbone of Enterprise software, Backend systems, and Android development. With regular updates (Java 17, 21+), it continues to be top-tier.
