Inheritance in Java: Building Hierarchical and Reusable Code

Inheritance is the cornerstone of reusability in Java. It allows a new class to absorb the fields and methods of an existing class, facilitating the "Is-A" relationship. Core Concept: Inheritance is a mechanism in which one object acquires all the properties and behaviors of a parent object. It is an integral part of OOPs (Object Oriented Programming System). The idea behind inheritance in Java is that you can create new classes that are built upon existing classes. When you inherit from an existing class, you can reuse methods and fields of the parent class. Moreover, you can add new methods and fields in your current class also.

1. The Syntax and Terminology

To implement inheritance, Java uses the extends keyword. This keyword signifies that you are creating a subclass that derives its functionality from a superclass.

  • Super Class (Parent): The class whose features are inherited.
  • Sub Class (Child): The class that inherits the other class. It can also add its own specific features.
  • Reusability: As the name specifies, reusability is a mechanism which facilitates you to reuse the fields and methods of the existing class when you create a new class.

Basic Code Example

class Employee {
float salary = 40000;
}

class Programmer extends Employee {
int bonus = 10000;

public static void main(String args[]) {
    Programmer p = new Programmer();
    System.out.println("Salary is: " + p.salary);
    System.out.println("Bonus is: " + p.bonus);
}
}

2. Types of Inheritance in Java

On the basis of class, there can be three types of inheritance in java: single, multilevel and hierarchical. In java programming, multiple and hybrid inheritance is supported through interface only.

Type Description Visual Structure
Single Inheritance One child class inherits from one parent class. Class A → Class B
Multilevel Inheritance A child class inherits from a parent, which is also a child of another parent. Class A → Class B → Class C
Hierarchical Inheritance Multiple child classes inherit from a single parent class. Class A → Class B & Class C

Why Multiple Inheritance is not supported in Java?

To reduce complexity and simplify the language, multiple inheritance is not supported in java. Consider a scenario where Class A has a method display() and Class B also has a method display(). If Class C inherits from both A and B, and you call display(), the compiler wouldn't know which method to execute. This is known as the Diamond Problem. To avoid this ambiguity, Java uses Interfaces to achieve similar functionality safely.

3. The Role of 'super' and Constructors

When you create an instance of a subclass, the parent class constructor is called first. This is known as Constructor Chaining. The super keyword is used to refer to the immediate parent class object.

  • It can be used to refer to immediate parent class instance variable.
  • It can be used to invoke immediate parent class method.
  • super() can be used to invoke immediate parent class constructor.

4. Comprehensive Interview Questions & Detailed Explanations

Q1. Does a subclass inherit the private members of its superclass?

No, a subclass does not inherit the private members of its parent class. However, if the superclass has public or protected methods (like getters and setters) for accessing these private fields, those methods can be used by the subclass.

Q2. What is the difference between IS-A and HAS-A relationship?

IS-A relationship: This is achieved through Inheritance. For example, a "Car IS-A Vehicle". It is a tight coupling.

HAS-A relationship: This is achieved through Association/Composition. For example, a "Car HAS-A Engine". It is used for better code modularity and flexibility.

Q3. Can we override a static method?

No, you cannot override a static method. If you define a static method with the same signature in the child class, it is called Method Hiding, not overriding. This is because static methods are bound to the class at compile-time, whereas overriding is a runtime phenomenon.

Q4. What is the 'final' class in inheritance?

If a class is declared as final, it cannot be extended by any other class. This is used to prevent inheritance for security reasons or to ensure the integrity of the class's behavior (e.g., the String class is final).

Q5. How does memory allocation work in Inheritance?

When an object of a subclass is created, memory is allocated for all the instance variables of the subclass as well as all the instance variables of its superclasses (all the way up to the Object class). This is why the child object has access to parent properties.

5. Advanced Theory: Upcasting and Downcasting

In Java, inheritance allows us to treat an object of a subclass as an object of its superclass. This is a key part of Polymorphism.

Upcasting: Casting a subtype to a supertype. It is done automatically by the compiler. Parent p = new Child();

Downcasting: Casting a supertype to a subtype. It must be done manually and can throw a ClassCastException if not handled carefully. Child c = (Child)p;

6. Summary of Key Rules

  • Constructors are NOT inherited but are called.
  • Private members are NOT inherited.
  • Final methods cannot be overridden.
  • Java supports single, multilevel, and hierarchical inheritance with classes.
  • The Object class is the root of the class hierarchy in Java.

7. Conclusion

Inheritance is a powerful tool that, when used correctly, can significantly reduce code redundancy and improve the logical structure of your software.
